#### Lue/katso/kuuntele ja tiivistä

##### Mastering Metasploit
- Penetraatiotesti etenee PTES-standardin (Penetration Testing Execution Standard) mukaan: Esivalmistelu (preinteractions) -> tiedustelu (reconnaissance) -> uhkamallinnus (Threat modeling) -> haavoittuvuusanalyysi (Vulnerability analysis) -> exploitaatio (Exploitation and post-exploitation) -> raportointi (Reporting)
- Tiedusteluvaihe vie 40–60 % ajasta –> mitä paremmin kohde tunnetaan, sitä tarkemmat hyökkäykset
- Virtuaaliympäristö (VMware/VirtualBox + Kali Linux) on suositeltava testiympäristö: snapshot-ominaisuus mahdollistaa palaamisen aiempaan tilaan
- Metasploit on Ruby-pohjainen kehys (luotu 2003), jota käytetään pääosin msfconsole-käyttöliittymällä
- Jos suora exploitti epäonnistuu palomuurin takia, vaihdetaan reverse shell -lähestymistapaan – kohde ottaa yhteyden hyökkääjään eikä toisin päin

##### Mitä 'nmap -sn' tekee?
- `nmap -sn` tekee **ping sweep** -skannauksen, joka selvittää mitkä hostit ovat verkossa ilman porttiskannausta.
- Se lähettää: ICMP echo request (perinteinen ping), TCP SYN porttiin 443 (koputetaan HTTPS-porttiin, että päästääkö palomuurin läpi) , TCP ACK porttiin 80 (koputetaan HTTP-porttiin eritavalla), ICMP timestamp request (kysyy kellonaikaa)
- Palauttaa vain mitkä IP-osoitteet vastaavat – ei tietoa porteista tai palveluista

**Huomioitavaa** Aiemmissa kotitehtävissä huomasin, että VMwaren yksityisessä verkossa -sn ei toimi kunnolla, koska se käyttää ARP-paketteja lähiverkossa eikä saa vastausta. Siksi tarvitaan komenttoon vielä `-Pn -n --send-ip`.

Lähteenä toimi (nmap.org/book/man-host-discovery.html).

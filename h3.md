##### Mastering Metasploit
- Penetraatiotesti etenee PTES-standardin (Penetration Testing Execution Standard) mukaan: Esivalmistelu (preinteractions) -> tiedustelu (reconnaissance) -> uhkamallinnus (Threat modeling) -> haavoittuvuusanalyysi (Vulnerability analysis) -> exploitaatio (Exploitation and post-exploitation) -> raportointi (Reporting)
- Tiedusteluvaihe vie 40–60 % ajasta –> mitä paremmin kohde tunnetaan, sitä tarkemmat hyökkäykset
- Virtuaaliympäristö (VMware/VirtualBox + Kali Linux) on suositeltava testiympäristö: snapshot-ominaisuus mahdollistaa palaamisen aiempaan tilaan
- Metasploit on Ruby-pohjainen kehys (luotu 2003), jota käytetään pääosin msfconsole-käyttöliittymällä
- Jos suora exploitti epäonnistuu palomuurin takia, vaihdetaan reverse shell -lähestymistapaan – kohde ottaa yhteyden hyökkääjään eikä toisin päin

##### Mitä 'nmap -sn' tekee?
- `nmap -sn` tekee **ping sweep** -skannauksen, joka selvittää mitkä hostit ovat verkossa ilman porttiskannausta.
- Se lähettää: ICMP echo request (perinteinen ping), TCP SYN porttiin 443 (koputetaan HTTPS-porttiin, että päästääkö palomuurin läpi) , TCP ACK porttiin 80 (koputetaan HTTP-porttiin eritavalla), ICMP timestamp request (kysyy kellonaikaa)
- Palauttaa vain mitkä IP-osoitteet vastaavat – ei tietoa porteista tai palveluista

**Huomioitavaa** Aiemmissa kotitehtävissä huomasin, että VMwaren yksityisessä verkossa -sn ei toimi kunnolla, koska se käyttää ARP-paketteja lähiverkossa eikä saa vastausta. Siksi tarvitaan komenttoon vielä `-Pn -n --send-ip`.

Lähteenä toimi (https://nmap.org/book/man-host-discovery.html) ja (https://nmap.org/book/host-discovery-controls.html).  Kyseessä on nmapin virallinen dokumentaatio eli ne on kirjoitettu nmapin omien kehittäjien toimesta, eli ei kolmannen osapuolen tulkintaa. Se on myös Primäärilähde eli menee suoraan alkuperäiseen lähteeseen eikä lueta jonkun muun tulkintaa siitä.

#### e) Vertaile nmap:n omaa tiedostoon tallennusta (-oA foo) ja db_nmap:n tallennusta tietokantoihin

##### nmap -oA foo
- Tallentaa tiedon kolmeen tiedostoon: **foo.nmap(teksti)**, **foo.xml(XML)** ja **foo.gnmap(grep-friendly)**
- Tiedostot ovat pysyviä ja säilyvät vaikka Metasploit suljetaan
- Niitä on helppo jakaa muille tai arkistoida muun muassa raportteja varten
- Ne voi avata millä tahansa tekstieditorilla
- XML-muoto sopii muihin työkaluihin
(Nmap, s.a.)

##### db_nmap (Metasploitin tietokanta)
- Tulokset tallentuvat automaattisesti PostgreSQL-kantaan
- Ne voidaan hakea ja suodattaa suoraan Metasploitissa (hosts, services, services -p 21)
- Exploitit löytävät kohteet automaattisesti kannan kautta (`setg RHOSTS`)
- Työskentely on nopeampaa kun ei tarvitse avata erillisiä tiedostoja
- Tukee workspaceja eli eri projektit pysyvät erillään
(Jaswal, 2020)

#### l) Attaaack!

MITRE ATT&CK on tietokanta, joka luokittelee hyökkääjien taktiikoita ja tekniikoita. Kävin läpi harjoituksessa tekemäni asiat ja yhdistin ne ATT&CK-tekniikoihin.

**Reconnaissance**: Ajoin nmap -sn ping sweepin löytääkseni verkossa olevat hostit, mikä on tekniikka Active Scanning: Scanning IP Blocks. Versiotunnistus db_nmap -sV:llä vastaa tekniikkaa Active Scanning: Vulnerability Scanning, koska tavoitteena oli löytää haavoittuva palveluversio eli vsftpd 2.3.4.

**Initial Access**: Murtauduin kohteeseen kahdella tavalla, molemmat ovat Exploit Public-Facing Application. Ensin vsftpd 2.3.4 -takaovella Metasploit-moduulilla, sitten suoraan bindshelliin komennolla nc 192.168.37.130 1524.

**Execution**: Kun pääsy oli saatu, ajoin komentoja shellissä. Tämä on Command and Scripting Interpreter: Unix Shell. Käytin sekä tavallista shelliä että Meterpreter-istuntoa.

**Credential Access**: Luin /etc/passwd ja /etc/shadow -tiedostot, joista löysin hashatut salasanat kaikille käyttäjille. Tämä on OS Credential Dumping: /etc/passwd and /etc/shadow). Salasanoja ei murrettu tässä harjoituksessa, mutta se olisi seuraava looginen vaihe.

**Discovery**: Keräsin tietoa ympäristöstä useilla komennoilla:

- `ifconfig` System Information Discovery: verkkoliitännät ja IP-osoitteet

- `ps aux` -> Process Discovery: käynnissä olevat prosessit, huomiona MySQL root-käynnistämänä

- `netstat -an` -> System Network Connections Discovery: avoimet portit ja yhteydet

- `cat /etc/passwd` -> Account Discovery: Local Account: järjestelmän käyttäjätilit

- Meterpreter sysinfo, ls, pwd -> File and Directory Discovery

Yhteensä harjoituksessa käytettiin viittä taktiikkaa ja kahdeksaa eri tekniikkaa. Varsinaista lateral movementia ei tehty, vaikka siihen tarvittavaa tietoa kerättiinkin.
